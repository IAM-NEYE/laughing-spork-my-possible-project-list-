This file contains example project entries to demonstrate how to structure your project ideas using the template format.

## Tips for Writing Good Project Descriptions

1. **Be Specific**: Instead of "social media app", write "photo-sharing app for pet owners"
2. **Include the Problem**: What problem does your project solve?
3. **Consider Your Audience**: Who would use this project?
4. **Be Realistic**: Choose technologies you know or want to learn
5. **Estimate Conservatively**: Projects usually take longer than expected
6. **Update Regularly**: Keep status and priorities current

## Status Definitions

- **Idea**: Just a concept, needs more planning
- **Planning**: Researching technologies, creating detailed specs
- **In Progress**: Actively working on the project
- **Paused**: Temporarily stopped work
- **Completed**: Project finished and deployed/released
- **Archived**: No longer pursuing this idea
